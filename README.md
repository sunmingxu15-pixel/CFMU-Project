## 基于条件流匹配的连续遗忘学习项目 (CFMU-Project)

### 项目概述

本项目专注于探索深度生成模型，特别是**条件流匹配 (Conditional Flow Matching, CFM)** 模型中的**机器学习反学习 (Machine Unlearning)** 技术。项目的核心目标是实现让一个预训练好的生成模型“忘记”其学习到的关于特定数据类别或数据分布中特定区域的知识，而无需在剔除相关数据后从头开始重新训练模型。

更进一步，本项目深入研究了**连续遗忘学习 (Continual Unlearning)** 的复杂场景。在这一场景下，模型需要按顺序、分阶段地忘记多个不同的数据类别，并有效避免在后续遗忘任务中发生“灾难性遗忘”——即重新生成了先前已被指令忘记的内容。

项目通过在经典的 `MNIST` 数据集和多个二维合成数据集（`Checkerboard`, `Circles`, `Moons`）上的大量实验，验证了所提出的方法的有效性。

### 核心思路与技术

1.  **条件流匹配 (Conditional Flow Matching, CFM):**
    项目采用CFM作为基础生成模型。CFM是一种强大而高效的生成模型训练方法，它通过学习连接数据分布与噪声分布的概率流（向量场）来生成数据。

2.  **能量加权反学习 (Energy-based Reweighted Flow Matching, ERFM):**
    在二维数据集的遗忘任务中，项目实现了一种基于能量函数的方法。通过定义一个“能量函数”来为需要遗忘的数据区域赋予高能量值，然后在训练过程中通过一个加权损失函数，降低模型生成这些高能量区域样本的可能性，从而达到“遗忘”的目的。

3.  **连续遗忘中的灾难性遗忘问题:**
    在连续遗忘任务中（例如，先忘记 MNIST 数字 '1'，再忘记 '3'，然后是 '5'），一个简单的“朴素”方法是每一步都在已遗忘类别集合上进行训练。实验发现，这种方法会导致模型性能衰退，先前遗忘的类别可能会被“部分记起”。

4.  **正交约束与选择性蒸馏 (Orthogonality Constraints & Selective Distillation):**
    为了解决上述问题，本项目在 `mnist Consecutive forgetting.ipynb` 中探索了一种高级的连续遗忘策略。该策略包含两个关键部分：
    *   **正交约束 (Orthogonality Loss):** 在训练模型忘记新类别时，约束其参数更新方向与之前遗忘任务的更新方向近似“正交”。这可以有效防止对历史遗忘任务的干扰。
    *   **选择性蒸馏 (Selective Distillation):** 强制模型在处理“保留类”数据时的行为与上一阶段的模型保持一致，极大地稳定了模型参数，从根本上防止了知识的灾难性遗忘。

### 实验内容

本项目包含以下几个核心实验：

*   **MNIST 单类别遗忘 (`mnist.ipynb`):**
    训练一个类别条件的CFM模型，并使其“忘记”如何生成数字 '1'。

*   **MNIST 连续多类别遗忘 (`mnist Consecutive forgetting.ipynb`):**
    实现朴素连续遗忘和基于“正交约束+选择性蒸馏”的先进连续遗忘方法，顺序遗忘数字 '1', '3', '5'，并对比了两种方法的效果。

*   **2D数据集区域遗忘 (`checkboard.ipynb`, `circles.ipynb`, `moons.ipynb`):**
    在三个经典的2D数据集上，训练CFM模型学习从高斯噪声到目标分布的映射，并使用ERFM方法遗忘目标分布的特定区域（如棋盘格的顶行和右列、双同心圆的外环等）。

*   **2D数据集连续区域遗忘 (`checkboard Consecutive forgetting.ipynb`, `checkboard Consecutive forgetting 2.ipynb`):**
    在棋盘格数据集上进行连续遗忘实验，例如先遗忘顶行，再遗忘右列，并探索了不同的遗忘顺序以及更高级的连续遗忘算法（如ROVER）。

所有实验都包含了丰富的可视化代码，能够动态地展示模型在学习和遗忘过程中的概率流轨迹和生成样本的分布变化。

### 如何运行

本项目代码均由 `Jupyter Notebook` 文件构成。您只需安装必要的Python依赖（如 `torch`, `torchdyn`, `torchcfm`, `matplotlib` 等），然后按顺序执行 `Notebook` 中的代码单元即可复现实验结果。

-   对于每个数据集，通常包含一个 `_train.ipynb` 或 `*.ipynb` 的第一部分用于模型的预训练。
-   后续的 `_unlearn.ipynb` 或 `Consecutive forgetting.ipynb` 文件则依赖于预训练好的模型权重，用于执行遗忘实验。
